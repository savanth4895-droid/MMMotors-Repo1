<product_requirements>
The user requested a full-stack application to manage a two-wheeler business in India, encompassing vehicle sales, insurance, services, and spare parts. The application needs a login page navigating to a dashboard. The dashboard should feature four main tiles: Sales, Services, Vehicles Stock, and Spare Parts.

**Module-specific requirements:**
*   **Sales:** Create Invoice (detailed form for customer/vehicle/insurance details, preview/download/print), View Invoices (search bar, table with Invoice No., Date, Customer Name, Vehicle Model, Amount, Status, Actions), Add Customer (View/Import/Export buttons, comprehensive customer/vehicle/insurance nominee form), View Customer Details (search bar, table with customer/vehicle details, Actions), Sales Report (monthly sales graph, brand-wise sales graph), Insurance Renewal (table with customer/vehicle details, expiry date, status, actions).
*   **Services:** New Service registration (form for customer/vehicle details), View Registration (search bar, table with registration details, actions), Job cards (search bar, add new job button, table with job card details, status, actions), Create Bills (detailed GST-compliant billing form), View Bills, Service Due.
*   **Vehicles Stock:** Separate clickable tiles for brands (TVS, BAJAJ, HERO, HONDA, TRIUMPH, KTM, SUZUKI, APRILIA). Clicking a tile opens a detailed page for that brand with vehicle specifics (Date, Chassis No, Engine No, Model, Color, Key no., Inbound/Outbound Location, Status, Page Number).
*   **Spare Parts:** Create Bills, View Bills, Inventory, Reports, Low stock.

Additional requirements included a multi-user system, basic implementation of all modules, vehicle tracking from stock to service history, and a clean, minimal, mobile-responsive design.
</product_requirements>
